# Docker Compose

The fastest way to run the stack — one compose file, two images, three commands.

## Files

| Path | What |
|---|---|
| `docker-compose.yml` | Top-level orchestration. |
| `backend/Dockerfile` | Node 20 alpine, multi-stage. |
| `frontend/Dockerfile` | Nginx alpine serving static + reverse-proxying `/api/*`. |
| `frontend/nginx.conf` | Same routes as the bare-metal vhost; upstream is the compose service `backend:8787`. |

## Bring up

```bash
git clone https://github.com/0xNoramiya/ctf-blockchain-infra.git
cd ctf-blockchain-infra

cp backend/.env.example         backend/.env
cp backend/challenges.example.json backend/challenges.json

# 1. edit backend/.env             (RPC_URL, FLAGS, SIGNER_KEYS)
# 2. edit backend/challenges.json  (paste your deployed contract addresses)

docker compose up -d --build
docker compose logs -f
```

The frontend is on `${PUBLIC_PORT:-8080}` of the host. Put Cloudflare / your reverse proxy in front of it.

## Smoke test

```bash
curl -s http://localhost:8080/api/health        # { ok: true, challenges: N }
curl -s http://localhost:8080/api/config | jq . # full manifest, minus secrets
```

## Day-to-day

| Action | Command |
|---|---|
| Edit manifest | `vim backend/challenges.json && docker compose restart backend` |
| Edit secrets | `vim backend/.env && docker compose restart backend` |
| Edit frontend | `vim frontend/*.{html,css,js} && docker compose build frontend && docker compose up -d frontend` |
| Tail logs | `docker compose logs -f backend` |
| Drop ZIPs for download | drop `chN.zip` into `frontend/dist/` — mounted read-only into nginx |

## What's in each image

**`backend`** (`ctf-backend:local`)

- Node 20 alpine
- Runs as user `app` (uid generated by `adduser -S`)
- Listens on `8787`
- Reads `/app/challenges.json` (bind-mounted from `./backend/challenges.json:ro`)
- Reads env from `./backend/.env` via `env_file:`
- Healthcheck: `GET /api/health` inside the container every 30s

**`frontend`** (`ctf-frontend:local`)

- nginx 1.27 alpine
- Listens on `8080`
- Serves `/` from `/usr/share/nginx/html` (the static files baked in)
- Reverse-proxies `/api/*` to `backend:8787` via docker-compose's `internal` bridge network
- Mounts `./frontend/dist:/usr/share/nginx/html/dist:ro` for player-distributable zips
- Healthcheck: `GET /healthz` inside the container

## Hardening in front

Compose only exposes the frontend port. The backend is reachable only via `frontend:8080/api/*` from outside.

Restrict the exposed port at the OS firewall to Cloudflare's CIDRs:

```bash
sudo PORT=8080 ./deploy/cloudflare/ufw-allow-cloudflare.sh
sudo ufw default deny incoming && sudo ufw allow ssh && sudo ufw --force enable
```

See [Cloudflare hardening](cloudflare.md).

## Why mount instead of bake?

`challenges.json` and `.env` are bind-mounted so iteration is fast (`docker compose restart backend` is ~1 second). The frontend's static files are baked in because they rarely change mid-event.

If you'd rather bake everything for immutable deploys:

1. Add `COPY challenges.json /app/` to `backend/Dockerfile`.
2. Drop the volumes block.
3. Tag and push images to your registry.

## Resource budget

| Service | Idle RAM | Peak RAM (event traffic) | CPU (idle / peak) |
|---|---|---|---|
| backend | ~70 MB | ~150 MB | <1% / spike on RPC calls |
| frontend (nginx) | ~10 MB | ~20 MB | negligible |

A 1 vCPU / 1 GB RAM box runs both with room to spare.
